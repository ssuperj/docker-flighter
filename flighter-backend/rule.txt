브랜치 네이밍 규칙
    main
    develop
    feature-(issue_number)
    release-(version_number)
    Hotfix-(Issue_number)

이미지 네이밍.
    컴포넌트 유형(접두사) + 이름 + 상태(접미사). png (확장자)
    Ex)
    btn-pirmary-normal.png
    ic-like-hover.png

깃 커밋 규칙
    (동사 영어) + (구체적으로 뭘했는지 한글 or 영어)
    Ex)
    git commit -m 'add navigation var'
    git commit -m 'add 네비게이션 바'
    git commit -m 'change 배경색 톤업'
